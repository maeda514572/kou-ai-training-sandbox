# Review Rubric

PRを見るときは、次の順番で確認します。

## 1. Issue適合

- Issueの目的を満たしているか
- DoDを満たしているか
- Issue外の変更がないか

## 2. 実装

- 変更が小さいか
- 読みやすいか
- 不要な抽象化がないか

## 3. テスト

- 必要なテストがあるか
- 回帰を防げるか
- `npm test` が通っているか

## 4. 安全性

- `.env*` や認証情報に触れていないか
- 本番APIや外部送信に触れていないか
- 金銭判断や契約変更に関わっていないか

## 5. マージ判断

- レビューコメントが解決済みか
- CIが通っているか
- 差分を説明できるか
